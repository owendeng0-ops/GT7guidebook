# GT7 Track Atlas V1.2 训练闭环路线

## 版本定位

V1.2 的目标是把 GT7 Track Atlas 从“赛道布局资料库 + 单条最佳圈速记录”推进到“手动记录驱动的日常训练闭环”。本版本不做 PS5 数据同步、不做账号系统、不引入后端，也不把重点放在公开发布能力上。

> 状态：V1.2.0 已在主站实装。本文件保留为产品与实现依据。

核心用户是进阶 GT7 玩家。核心闭环是：

```text
选择今天要练的 layout -> 设定目标 -> 记录本次练习 -> 复盘差距 -> 下次继续
```

## 产品目标

- 让用户打开首页后能立刻知道“今天该练什么”。
- 让每个官方 layout 都能沉淀多次练习记录，而不是只保存一个最佳圈速。
- 让目标圈速、车辆推荐、失误笔记和练习状态形成一个可复盘的工作台。
- 保持静态站和离线打开能力，继续使用 `localStorage` 保存用户数据。

## 非目标

- 不接入 PSN、GT7、第三方账号或云同步。
- 不做多人排行榜、评论区、公开分享页或服务端统计。
- 不重新扩展车辆/赛道官方数据源；V1.2 只增强使用现有数据的训练体验。
- 不引入 React、Vue、打包工具或数据库。

## V1.2 功能范围

### 1. 训练入口：今天练什么

首页训练仪表盘改为训练决策入口，优先展示：

- 当前训练 layout：最近被设为训练中的 layout。
- 最近一次练习：展示 layout、圈速、车辆和练习时间。
- 下一条推荐练习：基于推荐规则选出一个 layout。
- 未完成目标：展示未达标数量、本周练习次数和已达标数量。

推荐优先级固定为：

1. 状态为“练习中”的 layout。
2. 收藏但未达标的 layout。
3. 真实赛道 layout。
4. 同一地点下多布局差异明显的 layout。
5. 当前难度筛选匹配的 layout。

赛道索引中的状态从地点级摘要调整为“当前推荐 layout 状态”。每个赛道按钮仍显示地点名、中文名和布局数量，但状态徽标应优先反映该赛道内最需要继续练的 layout。

### 2. 详情页：训练工作台

当前 layout 的训练卡片改为工作台结构：

- 目标区：目标难度、目标圈速区间、目标中位、个人最佳、目标差距。
- 操作区：设为当前训练、标记已达标、收藏 layout、重置记录。
- 本次练习记录：圈速、车辆、轮胎/辅助设置、失误点、练习感受。
- 复盘区：最近 5 次练习记录，显示是否刷新最佳和与目标差距变化。

圈速指南从静态表格改为三段训练任务：

- 稳定完赛：优先连续 3 圈无重大失误，刹车点固定。
- 误差控制：单圈误差控制在 1 秒左右，开始调整线路和出弯节奏。
- 极限攻弯：追求分段极限，重点管理出弯速度、轮胎负载和车辆姿态。

### 3. 手动记录模型

继续读取旧 key：

```text
gt7-track-atlas-training-v1
```

新增 V1.2 key：

```text
gt7-track-atlas-training-v1.2
```

建议数据形态：

```js
{
  version: "1.2",
  layouts: {
    [layoutId]: {
      targetDifficulty: "新手" | "进阶" | "高手",
      status: "not-started" | "active" | "complete",
      favorite: boolean,
      notes: string,
      currentVehicle: string,
      currentSetup: string,
      updatedAt: string,
      sessions: [
        {
          id: string,
          lapSeconds: number,
          vehicle: string,
          setup: string,
          mistakes: string,
          feeling: string,
          createdAt: string
        }
      ]
    }
  }
}
```

兼容规则：

- 如果 V1.2 key 不存在，页面从 V1.1 记录生成默认 V1.2 视图，但不立即删除旧数据。
- 旧字段 `bestLapSeconds` 作为第一条历史记录或只读最佳来源保留。
- V1.2 的最佳圈速从 `sessions[].lapSeconds` 和旧 `bestLapSeconds` 中取最小值。
- 损坏 JSON、缺字段或未知状态必须回退到空记录，不影响页面加载。

### 4. 复盘与进步反馈

layout 内展示最近 5 次练习：

- 圈速，使用固定输入格式解析后的标准显示。
- 车辆和设置。
- 与目标中位的差距。
- 是否刷新个人最佳。
- 简短练习感受。

首页轻量统计：

- 本周练习次数。
- 已达标 layout 数。
- 最常练赛道。
- 最大进步 layout。

统计只使用本地数据，不上传、不跨设备同步。

### 5. 车辆推荐联动

车辆推荐卡片增加“适合当前目标”的训练标签：

- 稳定完赛：容错、转向稳定、刹车可控。
- 误差控制：低速牵引、出弯稳定、线路修正空间。
- 极限攻弯：高速稳定、刷圈效率、极限速度。

如果用户在当前 layout 记录过车辆：

- 详情页优先显示“上次使用车辆”。
- 车辆推荐区增加“推荐替代车辆”提示。
- 不新增外部车辆数据源；只基于现有推荐文案和车辆缩略图组织展示。

## 实施顺序

### Milestone 1：数据层与兼容

- 新增 V1.2 localStorage 读写函数。
- 从 V1.1 数据生成兼容视图。
- 增加 session 新增、删除、重置、最佳圈速计算函数。
- 保持旧训练状态、收藏和笔记继续生效。

### Milestone 2：训练工作台 UI

- 重构当前 layout 训练卡片。
- 增加本次练习记录表单。
- 增加最近 5 次练习记录列表。
- 保持圈速输入自动格式化，用户只输入数字。

### Milestone 3：首页训练决策

- 仪表盘改为“今天练什么”结构。
- 实现固定推荐优先级。
- 赛道索引状态改为推荐 layout 状态。
- 保留区域、难度、训练状态筛选。

### Milestone 4：复盘与车辆联动

- 增加本周练习、已达标、最常练、最大进步统计。
- 车辆卡片增加训练目标标签。
- 展示上次使用车辆和推荐替代车辆。

## 验收标准

### 本地记录

- 空 `localStorage`、旧 V1.1 记录、损坏 JSON 都不会导致页面报错。
- 同一 layout 连续录入 3 次练习后，刷新页面仍能看到最近记录、最佳圈速和最近练习时间。
- Nürburgring GP 与 Nordschleife 的练习记录完全独立，不串 layout。

### 训练闭环

- 收藏一个未达标 layout 后，仪表盘推荐优先出现该 layout。
- 标记已达标后，推荐自动转向下一条未达标 layout。
- 修改目标难度后，目标圈速、目标差距和训练任务同步变化。

### 交互与可用性

- 圈速输入继续支持只输入数字，自动显示为固定时间格式。
- 移动端训练表单、记录列表、操作按钮不溢出。
- `file:///E:/GT7guidebook/index.html` 直接打开仍可用。
